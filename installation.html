SparkBind — Installation Guide

Ishyiga Pharmacy Network · Distributed ETL Agent

Version 1.0-SNAPSHOT Java 21+ · Maven 3.9+

0 What is SparkBind?

SparkBind is a background sync agent that runs silently on each Ishyiga pharmacy computer. It reads data from the local Ishyiga POS database (Apache Derby) and sends it to the central Hirwa cloud server — every 30 seconds or on a schedule.

It syncs: stock levels, medications, patients, and sales invoices. It also backs up the local database to AWS S3 every 12 hours.

SparkBind runs in the background. It has no window or interface — it works silently and logs activity to a logs/ folder.

2 Build the Project

Open a new PowerShell window and run these commands one by one:

Step 2a — Set Java version for this session:

$env:JAVA_HOME = "C:\Program Files\Java\jdk-22"

Step 2b — Navigate to the project folder:

cd "C:\Users\TRAMED\Downloads\hirwa-spark-bind-develop\hirwa-spark-bind-develop"

Step 2c — Build:

& "C:\Users\TRAMED\maven\apache-maven-3.9.6\bin\mvn.cmd" clean package -DskipTests

Maven will download dependencies and compile the code. At the end you should see:

BUILD SUCCESS — Total time: ~30 seconds

The built file will be created at:

target\SparkBind-1.0-SNAPSHOT.jar

3 Configure the App

SparkBind reads its settings from a file called application.properties. You need to create this file by copying the example:

Step 3a — Copy the example config:

copy "src\main\resources\application.properties.example" "src\main\resources\application.properties"

Step 3b — Open and fill in the fields:

Open src\main\resources\application.properties in any text editor and fill in the blanks:

FieldWhat it meansWhere to get it
db.path= Path to the local Ishyiga Derby database folder on this machine Find the folder named VINEBENDB or similar on the hard drive. Example: C:/POS/data/VINEBENDB
db.backup.path= Where to save local DB backups (zip files) before uploading to S3 Any local folder. Example: C:/backups/ optional
api.base.url= The web address of the central Hirwa server Get from the Hirwa/Ishyiga technical team
api.key= Secret password that proves this pharmacy is allowed to send data Get from the Hirwa/Ishyiga technical team required
api.pharmacy.id= The unique ID for this specific pharmacy branch Get from the Hirwa/Ishyiga technical team required
inventory.api.url= URL endpoint for sending stock/inventory updates Get from the Hirwa/Ishyiga technical team
inventory.api.key= API key specifically for the inventory endpoint Get from the Hirwa/Ishyiga technical team
fetch.interval.seconds=30 How often (in seconds) to sync stock data Already set to 30. Change if needed.
graebm.enabled=false Whether to sync GRA tax (EBM) transactions Set to true only if this pharmacy uses GRA EBM system
aws.s3.bucket.name= AWS S3 bucket name for database backups Get from Hirwa team optional
You only need the fields marked required to get started. Optional fields can be left blank for now.

Example of a filled config:

# Database
db.url=jdbc:derby://localhost:1527/VINEBENDB
db.path=C:/POS/data/VINEBENDB
db.backup.path=C:/backups/

# Central API
api.base.url=https://api.hirwa.rw
api.key=your-secret-key-here
api.pharmacy.id=VINE-HQ

# Sync interval
fetch.interval.seconds=30

4 Run SparkBind

Once configured, you can start SparkBind in two ways:

Option A — Run once manually (for testing):

start.bat

This runs the sync once and shows output in the terminal. Good for testing your config is correct.

Option B — Install as a Windows auto-start service (recommended for production):

# Run PowerShell as Administrator, then:
setup.bat

This registers SparkBind as a Windows Scheduled Task that starts automatically every time the computer boots — no manual action needed.

To stop a running instance, create a file called shutdown.txt in the project folder. SparkBind will detect it and stop gracefully.

5 Verify It's Working

After starting, check these signs that SparkBind is working correctly:

  1. Terminal shows: Internet connection verified.
  2. Terminal shows: Database connection successful.
  3. Terminal shows: Sent Live Stocks: [...] every 30 seconds
  4. No ERROR lines appear in the output
  5. Log files appear in the logs/ folder inside the project
If you see No internet connection available — check your network. SparkBind will retry automatically every hour.
