✗ Known Code Bug — To Fix When Ready

Bug location: C:\SparkBind\src\main\java\com\hirwa\PharmacyDistributedJob.java — lines 113–120
Impact: start.bat will crash on the database connection even after credentials are added.

What the bug is:

PharmacyDistributedJob (used by start.bat) opens the Derby database in embedded mode using the folder path (db.path). But VSDC already holds that database locked in embedded mode — so SparkBind gets error XSDB6: Another instance of Derby already has the database locked.

The fix is one line — use db.url (network mode, already working) instead of db.path (embedded mode, blocked):

// CURRENT CODE (broken — embedded mode, crashes with XSDB6):
private static DerbyClient createDerbyClient(String dbPath) {
    return new DerbyClient() {
        public Connection connect() throws SQLException {
            return DriverManager.getConnection("jdbc:derby:" + dbPath);
        }
    };
}

// FIXED CODE (uses network mode, same as what already works):
private static DerbyClient createDerbyClient(String dbPath) {
    return new DerbyClient() {
        public Connection connect() throws SQLException {
            return DriverManager.getConnection(AppConfig.getProperty("db.url"));
        }
    };
}
After applying this fix, rebuild with mvn clean package -DskipTests and the JAR will use the network connection that already works.

i Two Sync Modes — Know the Difference

SparkBind has two separate programs that sync different data:

How to run Program What data it syncs Credentials it needs
java -jar SparkBind.jar Main.java Live stock levels (every 30 seconds) inventory.api.url, inventory.api.key, secret.key, hmac.algorithm
start.bat PharmacyDistributedJob.java Products, Patients, Sales invoices (every 1 hour) api.base.url, api.key, api.pharmacy.id
Both need to run together for a full sync. Main.java handles real-time stock. PharmacyDistributedJob handles the daily data (medications, patients, sales).

i What Hirwa Is — The Big Picture

Hirwa is a central data warehouse built to collect sales, inventory, and patient data from ~500 Ishyiga pharmacies across Rwanda. Each pharmacy runs SparkBind locally — it reads from the local Ishyiga POS Derby database and pushes data to the central Hirwa API (Spring Boot + PostgreSQL in the cloud).

The central database uses a Star Schema (Kimball design) with dimension tables (dim_product, dim_client, dim_branch) and fact tables (fact_sales, fact_inventory). This allows cross-pharmacy reports like:

  • Top selling medications across all branches this month
  • Stock expiring in Pharmacy A but in high demand at Pharmacy B → trigger internal transfer
  • A patient's medication history even if they visit different pharmacies
  • Price consistency checks across 500 locations
Vine Pharmacy is already connected (4 branches: kh, hq, kf, kacyiru). TRAMED Nyarugenge is the next branch to be onboarded.
