Phase 1 — Planning

Project scope, objectives, stakeholders, and technology decisions

🎯 Project Overview

Pharmacie Tramed (KN 151 St, Kigali, Rwanda) is undergoing digital transformation via a 4-month CS internship project. The system builds an AI intelligence layer on top of the existing Ishyiga POS — rather than replacing Ishyiga (which handles billing), this adds demand forecasting, customer retention workflows, a loyalty programme, a public marketplace, and an AI chatbot assistant.

Key Design Decisions

WHY SQLITE?

Single-location pharmacy. Zero-config, embedded, easy to back up (one file). Sufficient for <100 concurrent users.

WHY WA.ME NOT WHATSAPP API?

WhatsApp Business API requires approval and monthly cost. wa.me deep links work instantly, zero cost, zero approval required.

WHY LIFETIME POINTS FOR TIER?

Separating points_balance (spendable) from lifetime_points (tier gate) means customers never lose their tier when redeeming.

WHY HAIKU NOT SONNET?

Customer service queries are simple (stock checks, pricing). Haiku is 10x cheaper and fast enough for pharmacy assistant use.

📊 Forecasting Algorithm (forecasting.py)

5-step ML pipeline, runs per product on each forecast page load:

1
Aggregate daily sales

Query sale_items JOIN sales for last 90 days, group by date, fill zero-sales days → DataFrame with date, qty, day_index, dow

2
Day-of-week multipliers

For each weekday 0–6: multiplier = mean(that_day_qty) / overall_mean. Captures weekly seasonality (e.g. busier Mondays).

3
Deseasonalise + linear trend

Last 30 days: y_deseas = y / dow_multiplier[dow] → numpy.polyfit(x, y_deseas, 1) → slope + intercept

4
Project forward 30 days

For each future day: predicted = max(0, trend(future_index) × dow_multiplier[dow])

5
Reorder recommendation

days_remaining = stock / avg_daily. Thresholds: critical ≤3d, urgent ≤10d, soon ≤14d. Reorder qty = forecast_14d + 7×avg_daily − stock

🤖 AI Chat (chat.py)

Every POST /api/chat call rebuilds the system prompt with live inventory data:

System prompt includes:
  - Pharmacy name + address
  - Full live inventory: name, category, price, stock status per product
  - Guidelines: multilingual (English/French/Kinyarwanda), no diagnosis,
    suggest alternatives if out of stock, direct orders to WhatsApp

Model: claude-haiku-4-5-20251001
Max tokens: 512
Pattern: multi-turn (messages array passed from frontend JS)

⚠ Falls back gracefully to {error: "Chat unavailable"} if ANTHROPIC_API_KEY is not set. Frontend shows WhatsApp fallback message.

Gap 3.1 — Data exists but no insights are extracted

Ishyiga records every sale, but there is no system to transform that data into meaningful insights. No dashboard, no trend analysis, no forecasting — the data sits completely unused.
There is no custom reporting dashboard. Revenue, margins, and top products are not visible at a glance.
